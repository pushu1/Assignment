5) What the deference between undefined and undeclare in JavaScript?

Undeclared

When a variable is “undeclared,” it means that it has not been declared or defined in the current scope. In other words, it hasn’t been given a value or a type. This can happen if you try to use a variable without declaring it first, or if you misspell the variable name.

Undefined

Unlike “undeclared” variables, “undefined” variables have been declared in the current scope, but have not been given a value. In other words, they have been initialized, but their value is not defined.

50) What is the drawback of declaring methods directly in JavaScript objects?

1. Memory Inefficiency:

  • Each instance of the object will have its own copy of the methods, even if they are identical.
  • This can lead to increased memory usage, especially when dealing with many instances of similar objects.

2. Performance Overhead:

  • Creating new method copies for each object instance takes time and resources.
  • This can impact performance, particularly in applications with frequent object creation and method calls.

3. Lack of Inheritance:

  • Methods declared directly within objects are not shared through inheritance.
  • This means each object must redefine its own methods, even those that are common across multiple objects.
  • Prototype-based inheritance, on the other hand, allows sharing methods efficiently among related objects.

4. Potential for Naming Conflicts:

  • If multiple objects with the same method names are combined, their methods might overwrite each other, leading to unexpected behavior.

5. Issues with this Keyword:

  • When a method is called directly within an object, this refers to the global object, not the object itself.
  • This can lead to errors and unexpected behavior if not handled carefully.

The Browser Object Model (BOM) in JavaScript includes the properties and methods for JavaScript to interact with the web browser. Array

when to use array:

We use arrays whenever we want to create and store a list of multiple items in a single variable. Arrays are especially useful when creating ordered collections where items in the collection can be accessed by their numerical position in the list. Just as object properties can store values of any primitive data type (as well as an array or another object), so too can arrays consist of strings, numbers, booleans, objects, or even other arrays.
